# 手持消防喷头与消防栓系统规划

本文档记录“消防栓固定端 + 玩家手持消防喷头端”的完整设计。该系统暂不实现，仅作为后续开发依据。

## 目标

- 添加一个可放置的消防栓方块，作为固定供液端。
- 添加一个“手持喷头控制器”物品，玩家可从消防栓中取出并绑定到消防栓。
- 玩家手持已绑定控制器时，左键触发喷射，并屏蔽原版破坏与攻击行为。
- 喷射时在消防栓与玩家手持喷头之间渲染一条水带。
- 控制器可被丢出为中立实体，并在地面保留水带连接渲染。
- 消防栓被破坏、玩家死亡、切换手持槽、控制器实体失效等情况必须可靠解除绑定。

## 非目标

- 本系统不直接复用现有 Fire Hose 双端端口连接逻辑。
- 本系统第一阶段不考虑 Create 动态结构上的完整运行。
- 本系统第一阶段不考虑 Sable 子结构上的特殊交互。
- “手持喷头控制器”针对不同喷头的模型后续再补充。

## 核心对象

### Fire Hydrant Block / Block Entity

消防栓固定端，负责：

- 保存 1B 流体缓存。
- 保存 3 个物品槽。
- 保存 A/B 模型状态。
- 管理当前手持控制器绑定。
- 为手持喷头提供流体。
- 方块破坏时掉落所有槽位物品，并立即解除所有绑定。

建议内部保存字段：

- `hydrantId`：消防栓唯一标识，首次创建后持久保存。
- `modelType`：箱式 / 桩式。
- `fluidTank`：容量 1000 mB。
- `hoseOrBucketSlot`。
- `nozzleSlot`。
- `controllerSlot`。
- `activeBinding`。

`activeBinding` 建议至少包含：

- 绑定 ID。
- 绑定玩家 UUID，可为空。
- 绑定实体 UUID，可为空。
- 当前绑定状态：玩家手持 / 地面实体 / 已清理。

### Handheld Nozzle Controller Item

手持喷头控制器物品存在两种状态：

- 未绑定状态：普通物品，可以放入消防栓 UI 的控制器槽。
- 已绑定状态：记录消防栓维度、位置、`hydrantId` 和绑定 ID。

绑定校验必须同时检查：

- 维度一致。
- 坐标处仍然是消防栓方块实体。
- 消防栓 `hydrantId` 与控制器记录一致。
- 消防栓当前 `activeBinding` 与控制器绑定 ID 一致。

不能只依赖方块坐标，否则消防栓被破坏后同位置重放可能导致旧控制器误连。

### Handheld Nozzle Controller Entity

玩家丢出已绑定控制器时，不使用普通 `ItemEntity` 表示绑定状态，而是生成专用实体。

实体行为：

- 在地面静止。
- 保留与消防栓之间的水带连接渲染。
- 玩家右键实体时尝试重新拾取为已绑定控制器。
- 实体死亡或存在时间超过 30 秒时：
  - 清理消防栓端绑定。
  - 删除实体。
  - 掉落未绑定的手持喷头控制器物品。

这样可以避免漏斗、磁铁、背包类模组直接拾取绑定物品导致状态失控。

### Handheld Hose Renderer

水带渲染层只负责视觉：

- 消防栓到玩家手持喷头。
- 消防栓到地面控制器实体。

该渲染层不参与真实 Fire Hose 端口连接，也不承担流体传输逻辑。

## 消防栓方块状态与模型

消防栓有两个模型：

- A 模型：箱式模型，资源目录为 `D:\CreateAdd\assets\消防模组\消防栓`。
- B 模型：桩式模型，资源目录同上。

建议 BlockState：

- `model_type`：`box` / `post`。
- `facing`：模型朝向。
- `waterlogged`：如需要水浸支持再添加。

模型 A：

- 尺寸为 16 宽、16 高、7 长。
- 允许从紧贴其它方块的 16x16 面输入流体。
- 允许从四个 16x7 面输入流体。

模型 B：

- 允许底部输入流体。

两种模型都应允许缓存被抽取。

## UI 与槽位

玩家 Shift + 右键消防栓时打开 UI。

UI 内容：

- 一个开关按钮，用于切换 A/B 模型。
- 水带 / 水桶槽位。
- 喷头槽位。
- 手持喷头控制器槽位。

槽位规则：

- 水带 / 水桶槽位：
  - 可放入消防水带。
  - 可放入空桶。
  - 三个槽位互不干涉。
- 喷头槽位：
  - 可放入任意喷头。
  - 建议后续通过接口或 tag 扩展可用喷头。
- 手持喷头控制器槽位：
  - 只能放入未绑定的手持喷头控制器。
  - 已绑定控制器不允许放入，避免状态混乱。

消防栓方块被破坏时：

- 立即解除所有绑定。
- 掉落三个槽位内所有内容物。
- 流体缓存默认丢失。

## 流体缓存与桶处理

消防栓携带 1B 流体缓存。

流体输入限制：

- 只允许输入喷头可以接受的流体。
- 建议复用当前喷头系统的可喷射流体判定，避免维护两套白名单。

桶填充逻辑：

- 当水带 / 水桶槽位内容物为空桶。
- 且消防栓缓存中有至少 1000 mB 流体。
- 且当前流体存在对应桶物品。
- 服务端将空桶替换为对应流体桶，并消耗 1000 mB。
- 如果当前流体没有桶物品，则不操作。

## 基础交互优先级

消防栓右键建议按以下顺序处理：

1. 玩家空手、非 Shift、槽位满足条件时：
   - 尝试取出一个已绑定的手持喷头控制器。
   - 成功后不打开 UI。
2. 玩家 Shift + 右键：
   - 打开 UI。
3. 玩家其它普通右键：
   - 默认不取控制器，交给物品自身或后续扩展。

取出绑定控制器的条件：

- 水带 / 水桶槽位内是消防水带。
- 喷头槽位内有合法喷头。
- 控制器槽位内有未绑定手持喷头控制器。
- 消防栓当前没有已有有效绑定。
- 玩家背包可以接收物品。

成功后：

- 控制器槽位减少 1 个控制器。
- 玩家获得一个已绑定控制器。
- 消防栓写入 `activeBinding`。
- 水带和喷头仍保留在消防栓槽位中。

## 手持喷头喷射输入

玩家主手持有已绑定的手持喷头控制器时：

- 左键不再破坏方块。
- 左键不再攻击实体。
- 左键改为喷射指令。

需要同时处理客户端和服务端：

- 客户端拦截左键，避免本地破坏动画和攻击动作。
- 客户端向服务端发送喷射开始 / 结束状态。
- 服务端再次校验绑定状态、消防栓状态和槽位内容。
- 服务端决定是否实际喷射、消耗流体并同步效果。

喷射条件：

- 控制器绑定仍有效。
- 消防栓方块仍存在。
- `hydrantId` 匹配。
- 消防栓内仍有对应水带和喷头。
- 消防栓缓存或输入链路能提供可喷射流体。

喷射行为：

- 喷射来源为玩家手持喷头位置。
- 喷射方向为玩家视线方向。
- 喷射效果复用现有喷头逻辑。
- 喷头类型由消防栓喷头槽决定。

## 喷射反作用力

手持喷头成功喷射时，给玩家一个与视线方向相反的小推力。

规则：

- 方向为 `-player.getLookAngle()`。
- 只有实际喷出流体时才施加推力。
- 玩家蹲下时不施加推力。
- 玩家趴下 / 爬行时不施加推力。
- 推力大小建议作为配置项，例如 `handheldNozzleRecoilStrength`。

该规则用于表现水流反冲，同时允许玩家通过蹲下或趴下降低后坐力。

## 绑定清理规则

以下事件必须解除绑定：

- 消防栓方块被破坏。
- 玩家死亡。
- 玩家切换手持槽，当前主手不再持有对应已绑定控制器。
- 玩家丢出已绑定控制器。
- 控制器实体死亡。
- 控制器实体存在时间超过 30 秒。
- 绑定校验失败。

清理时：

- 消防栓端 `activeBinding` 清空。
- 玩家物品栏中的对应已绑定控制器变回未绑定。
- 地面控制器实体解除绑定，后续只掉落未绑定控制器。
- 消防栓内水带和喷头槽位不变。

消防栓被破坏是最高优先级清理事件：

- 服务端进入移除流程时立即解除所有绑定状态。
- 然后再掉落槽位物品。
- 这样即使同位置重新放置消防栓，旧控制器也不会重新绑定到新方块。

## 丢出控制器行为

玩家丢出已绑定控制器时：

- 阻止生成普通绑定 `ItemEntity`。
- 从玩家物品栏移除该控制器。
- 生成 `HandheldNozzleControllerEntity`。
- 实体携带绑定信息和控制器显示信息。
- 消防栓端绑定状态从“玩家手持”切换为“地面实体”。

玩家右键地面控制器实体时：

- 校验消防栓绑定仍有效。
- 如果有效：
  - 删除实体。
  - 给玩家一个已绑定控制器。
  - 消防栓端绑定状态切回“玩家手持”。
- 如果无效：
  - 删除实体。
  - 掉落未绑定控制器。

实体超时或死亡时：

- 清理消防栓绑定。
- 删除实体。
- 掉落未绑定控制器。

## 网络同步

建议网络包：

- 客户端到服务端：手持喷头开始喷射。
- 客户端到服务端：手持喷头停止喷射。
- 服务端到客户端：绑定状态同步。
- 服务端到客户端：水带渲染目标同步。
- 服务端到客户端：喷射效果同步，或复用现有喷头粒子同步。

服务端必须是权威端：

- 客户端不能直接决定喷射成功。
- 客户端不能直接决定绑定有效。
- 客户端不能直接消耗消防栓流体。

## 渲染规划

需要渲染的内容：

- 玩家手持喷头模型。
- 消防栓到玩家手持喷头的水带。
- 消防栓到地面控制器实体的水带。
- 地面控制器实体模型。

控制器针对不同喷头的模型后续再提供。

水带渲染建议复用现有 Fire Hose 视觉曲线能力，但不要复用其连接状态协议。

## 兼容与风险

主要风险：

- 绑定物品复制。
- 消防栓破坏后旧控制器误连新方块。
- 玩家死亡或切换手持槽后绑定残留。
- 普通掉落物被其它模组拾取导致绑定状态丢失。
- 客户端左键拦截失败导致破坏动画或攻击残留。

防护策略：

- 使用 `hydrantId` 和 `bindingId` 双重校验。
- 丢出已绑定控制器时使用专用实体，不使用普通绑定掉落物。
- 消防栓破坏时立即主动清理所有绑定。
- 服务端定期或事件驱动校验 activeBinding。
- 所有实际喷射和流体消耗都由服务端决定。

## 建议配置项

- `handheldNozzleRecoilStrength`：手持喷头反作用力。
- `handheldNozzleEntityLifetimeTicks`：丢出实体存在时间，默认 600 tick。
- `handheldNozzleMaxDistance`：手持喷头与消防栓最大工作距离，如后续需要限制。
- `handheldNozzleFluidCapacity`：消防栓缓存容量，默认 1000 mB，第一阶段可固定。

## 实现阶段

### 第一阶段：消防栓基础

- 注册消防栓方块、方块实体、物品。
- 实现 A/B 模型状态和 UI 开关。
- 实现 3 个槽位。
- 实现 1B 流体缓存。
- 实现桶填充逻辑。
- 实现方块破坏掉落槽位物品。

### 第二阶段：控制器绑定

- 注册手持喷头控制器物品。
- 实现未绑定 / 已绑定数据。
- 空手右键消防栓取出绑定控制器。
- 实现 `hydrantId` 与 `bindingId` 校验。
- 消防栓破坏时解除绑定。

### 第三阶段：绑定生命周期

- 玩家死亡清理。
- 玩家切换手持槽清理。
- 绑定失效清理。
- UI 禁止在取用控制器时打开。

### 第四阶段：丢出实体

- 注册手持喷头控制器实体。
- 拦截已绑定控制器丢出行为。
- 实体右键重新拾取。
- 实体超时或死亡时清理绑定并掉落未绑定控制器。

### 第五阶段：手持喷射

- 客户端拦截左键。
- 服务端处理喷射开始 / 停止。
- 复用现有喷头效果逻辑。
- 消耗消防栓缓存或输入流体。
- 实现喷射反作用力。

### 第六阶段：渲染

- 渲染玩家手持喷头模型。
- 渲染消防栓到玩家的水带。
- 渲染消防栓到地面实体的水带。
- 后续接入不同喷头对应控制器模型。

## 待确认事项

- 手持喷头最大工作距离是否需要限制。
- 手持喷头喷射是否允许消防栓缓存为空但外部管道持续输入时直接工作。
- 消防栓缓存是否只作为缓冲，还是必须先填入缓存再喷射。
- 喷射时玩家是否需要播放专用动作或音效。
- 控制器实体是否允许被其它玩家拾取。
- 控制器是否允许副手使用。
- 消防栓 A/B 模型切换时是否保留当前朝向。
